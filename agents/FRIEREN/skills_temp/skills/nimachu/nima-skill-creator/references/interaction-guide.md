# 交互式设计指南 (Interactive Design Guide)

## 核心原则 (Core Principles)

### 1. 中文优先 (Chinese First)
- 用户交互全部使用中文
- 技术文档使用英文
- 桥接用户需求和技术实现

### 2. 渐进式披露 (Progressive Disclosure)
- 每次只问必要的问题
- 根据用户回答调整后续问题
- 避免信息过载

### 3. 简洁明确 (Clear and Concise)
- 问题简单直接
- 示例清晰易懂
- 避免技术术语

## 交互模式 (Interaction Patterns)

### Pattern 1: 选项式提问 (Multiple Choice)

```
问题: "你希望 Claude 帮你做什么事情？"
选项:
- "处理文件 (比如 PDF、Excel、图片等)"
- "帮我写东西 (比如文档、代码、报告)"
- "连接某个服务 (比如发消息、查数据)"
- "其他 (我来描述)"
```

**适用场景**: 需求初期探索

### Pattern 2: Confirm-then-Act

```
问题: "我理解的是...，对吗？"
选项:
- "对，没错"
- "不太对，我来补充"
- "完全不对，我重新说"
```

**适用场景**: 关键决策点确认

### Pattern 3: 细节深化 (Deep Dive)

```
问题: "关于 [具体细节]，你希望..."
选项:
- "详细说明 (我来描述)"
- "默认设置即可"
- "跳过这个"
```

**适用场景**: 已知方向后的细节确认

## 最佳实践 (Best Practices)

### 1. 问题设计
- ✅ 问题简短 (<20字)
- ✅ 选项互斥
- ✅ 包含"其他"选项
- ❌ 避免多重问题

### 2. 流程控制
- ✅ 每个阶段 2-3 个问题
- ✅ 重要决策点确认
- ✅ 允许用户回退
- ❌ 避免问题堆积

### 3. 错误处理
- ✅ 提供具体示例
- ✅ 解释为什么需要这个信息
- ✅ 允许用户跳过非关键问题

## 常见问题模板 (Common Templates)

### 技术方案咨询

```
问题: "实现这个功能，我想到两个方案："
选项:
- "方案A: [用简单语言描述]"
- "方案B: [用简单语言描述]"
- "我有其他想法"
```

### 运行环境确认

```
问题: "这个 Skill 你想在哪里用？"
选项:
- "只在当前这个项目用"
- "所有项目都能用"
```

### 资源确认

```
问题: "你有什么现成的资源需要包含吗？"
选项:
- "有代码/脚本"
- "有文档/说明"
- "有模板/素材"
- "不需要"
```

## 避免的模式 (Anti-Patterns)

### ❌ 问太多问题
- Wrong: 一次性问 10 个问题
- Right: 每次 2-3 个，分阶段进行

### ❌ 技术术语
- Wrong: "你希望使用 REST API 还是 GraphQL？"
- Right: "方案A: 直接读取文件 / 方案B: 连接在线服务"

### ❌ 模糊选项
- Wrong: "你感觉怎么样？"
- Right: "速度快(✅) / 质量高(✅) / 操作简单(✅) / 其他"

### ❌ 忽略用户反馈
- Wrong: 固定流程，不调整问题
- Right: 根据用户回答动态调整

```markdown
# PDF Processing Skill

## 1.1 核心 I/O 洞察

**问题**: "你希望 Claude 帮你做什么事情？"

**选项**:
- "处理文件 (比如 PDF、Excel、图片等)"
- "帮我写东西 (比如文档、代码、报告)"
- "连接某个服务 (比如发消息、查数据)"

- "PDF 文件 (文档、表单)"

- "图片 (JPG、PNG 等)"
- "其他类型"

---

**问题**: "你期望得到什么输出？"

**选项**:
- "提取文本和表格"
- "转换格式"
- "编辑/修改内容"
- "其他需求"
```
