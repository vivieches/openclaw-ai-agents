# WebSocket Listener — Cross-Platform Deployment Guide

## Overview

The WebSocket listener is a persistent background process that receives molt-message WebSocket push notifications, classifies messages by routing rules, and forwards them to local webhook endpoints (agent / wake dual mode).

## Platform Support

| Platform | Backend | Service File |
|----------|---------|-------------|
| **macOS** | launchd (LaunchAgent) | `~/Library/LaunchAgents/com.awiki.ws-listener.plist` |
| **Linux** | systemd (user unit) | `~/.config/systemd/user/awiki-ws-listener.service` |
| **Windows** | Task Scheduler (schtasks) | `%LOCALAPPDATA%\awiki-ws-listener\run-listener.bat` |

All platforms share the same CLI interface — `ws_listener.py install/uninstall/start/stop/status`.

## Files

| File | Description |
|------|-------------|
| `listener.example.json` | Routing rules configuration example — copy and customize |

## Quick Start

### 1. Prepare Configuration

```bash
cp service/listener.example.json service/listener.json
# Edit listener.json — set webhook_token, whitelist_dids, etc.
```

### 2. Install and Start

```bash
python scripts/ws_listener.py install --credential default --config service/listener.json
```

### 3. Verify

```bash
python scripts/ws_listener.py status
```

## Management Commands

```bash
# Install and start the service
python scripts/ws_listener.py install --credential default --mode smart

# Install with a custom config file
python scripts/ws_listener.py install --credential default --config service/listener.json

# Check service status
python scripts/ws_listener.py status

# Stop the service
python scripts/ws_listener.py stop

# Start a stopped service
python scripts/ws_listener.py start

# Uninstall (stop + remove)
python scripts/ws_listener.py uninstall

# Run in foreground for debugging (all platforms)
python scripts/ws_listener.py run --credential default --mode smart --verbose
```

## Log Locations

| Platform | Log Directory |
|----------|--------------|
| **macOS** | `~/Library/Logs/awiki-ws-listener/` |
| **Linux** | `~/.local/state/awiki-ws-listener/logs/` (also: `journalctl --user -u awiki-ws-listener.service`) |
| **Windows** | `%LOCALAPPDATA%\awiki-ws-listener\logs\` |

The `status` command output includes the log path for the current platform.

## Platform-Specific Notes

### macOS

- Service auto-starts on login (`RunAtLoad`) and auto-restarts on crash (`KeepAlive`).
- The plist is dynamically generated by `install` — no manual editing needed.

### Linux

- Uses `systemd --user` units with `Restart=on-failure` and `RestartSec=10`.
- For headless servers (SSH-only, no GUI session), enable lingering so the service starts at boot:
  ```bash
  sudo loginctl enable-linger $USER
  ```

### Windows

- Uses Task Scheduler with `ONLOGON` trigger and `LIMITED` privilege level.
- A wrapper `.bat` file is generated automatically.

## Troubleshooting

| Symptom | Solution |
|---------|----------|
| `status` shows not running | Check logs (path shown by `ws_listener.py status`) |
| JWT errors in logs | Refresh JWT: `python scripts/setup_identity.py --load default` |
| 401 from webhook | Verify `webhook_token` matches OpenClaw `hooks.token` |
| Webhook not receiving | Verify OpenClaw is running and the endpoint is reachable |
| Want to change mode | Uninstall, then reinstall with the new `--mode` |
| Linux: service not starting at boot | Run `sudo loginctl enable-linger $USER` |
